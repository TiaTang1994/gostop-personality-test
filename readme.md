## 2. 简介

### JsPsych和Flexbox的背景信息

JsPsych是一个用于设计和运行在线心理学实验的JavaScript库。它提供了丰富的插件和功能，使得实验设计变得简单高效。Flexbox是CSS的一种布局模式，能够轻松实现复杂的响应式布局。

### 为什么选择这些工具进行实验设计

JsPsych提供了强大的实验控制和数据收集能力，适合各种心理学实验。Flexbox则简化了页面布局和样式设计，使得实验界面更美观和用户友好。

## 4. JsPsych实验设计

-   欢迎页面
-   信息收集
-   个性测评
-   行为实验指导

行为实验部分设计了20次试次，包括10次“GO”信号和10次“STOP”信号。每个试次包括准备阶段、信号呈现和反应阶段。

### 插件的使用和自定义
使用了以下JsPsych插件：
-   `html-button-response`：用于显示欢迎页面和结束页面。
-   `survey-html-form`：用于信息收集。
-   `survey-multi-choice`：用于个性测评。
-   `html-keyboard-response`：用于行为实验和信号响应。

## 5. 数据收集和分析

### JsPsych的数据收集机制

-   **实验初始化**：在`jsPsych.init()`函数中，可以设置实验的参数和配置，包括数据收集的选项。
-   **数据属性**：在实验的每个阶段（称为trial），可以设置特定的数据属性，这些属性将被记录到数据集中。
-   **数据存储**：JsPsych使用内部的数据结构来存储实验数据。数据可以通过`jsPsych.data.get().json()`获取，并以JSON格式输出。

### 数据导出
-   **控制台输出**：在实验结束时，可以通过`console.log(data)`将数据打印到浏览器的控制台。
-   **页面显示**：使用`jsPsych.data.displayData()`方法可以直接在页面上显示数据。
-   **保存为文件**：可以将数据转换为JSON字符串，并使用`Blob`和`a[download]`元素允许用户下载数据文件。

### 基本分析方法
#### 个性测试部分
 1. 本测试内容为外向性，即个体神经系统的强弱和动力特征。
 2. 第5题和15题为反向计分，其余均为正向计分。

 4. 结果解释：
**外倾性（extraversion）/外向性**表示人际互动的数量和密度、对刺激的需要以及获得愉悦的能力。这个维度将社会性的、主动的、个人定向的个体和沉默的、严肃的、腼腆的、安静的人作对比。 
这个方面可由两个品质加以衡量：人际的卷入水平和活力水平。前者评估个体喜欢他人陪伴的程度，而后者反映了个体个人的节奏和活力水平。  
外向的人喜欢与人接触，充满活力，经常感受到积极的情绪。他们热情，喜欢运动，喜欢刺激冒险。在一个群体当中，他们非常健谈，自信，喜欢引起别人的注意。  
**内倾性（introversion）/内向性**的人比较安静，谨慎，不喜欢与外界过多接触。他们不喜欢与人接触不能被解释为害羞或者抑郁，这仅仅是因为比起外向的人，他们不需要那么多的刺激，因此喜欢一个人独处。内向人的这种特点有时会被人误认为是傲慢或者不友好，其实一旦和他接触你经常会发现他是一个非常和善的人。

#### 行为实验部分
1. 本测试主要用来评估和研究个体在接收到特定信号时的反应速度和准确性。
2.  测量内容：
- 反应时间（RT)：从接收到信号到做出反应所需的时间。
- 准确性：参与者正确执行任务的比例，即在“Go”信号时正确反应，在“Stop”信号时正确抑制反应。
- 抑制能力：在接收到“Stop”信号时抑制反应的能力，通常通过“Stop信号反应时间”（SST）来衡量，即从“Stop”信号出现到实际停止反应所需的时间。
3. 结果解释：
- 反应时间：较短的反应时间通常表示较高的处理速度，但也可能与冲动性有关。较长的反应时间可能表示更多的谨慎或处理速度较慢。
- 准确性：高准确性表明参与者能够有效地区分“Go”和“Stop”信号并做出适当的反应。低准确性可能表明参与者在任务执行上存在困难，或者存在注意力分散。
- 抑制能力：如果参与者在“Stop”信号后能够快速停止反应，这表明他们具有较好的抑制控制能力。如果“Stop”信号后的反应时间较长，可能表明抑制控制能力较弱。
4. 数据分析
分析结果时，研究者需要使用统计方法来比较不同条件或群体的表现。例如，比较不同年龄组、性别或患有特定认知障碍的个体在Go/Stop任务上的表现差异。

## 6. 常见问题解答（FAQ）


**问题1：如何确保数据的准确性？**

-   解决方案：在实验设计阶段，确保所有问题和指导都是清晰无误的。使用JsPsych的验证机制来检查参与者的输入是否有效。

**问题2：如何处理丢失的数据或不完整的实验？**

-   解决方案：在数据分析阶段，排除那些没有完成所有trials的数据集。或者，可以在实验设计中加入逻辑来检测不完整的响应，并提示参与者完成。

**问题3：如何提高实验的用户体验？**

-   解决方案：设计简洁直观的用户界面，确保实验流程清晰。提供明确的指示和反馈，以及在实验结束时的感谢信息。

**问题4：如何确保实验的随机性？**

-   解决方案：使用JsPsych的随机化功能，如`jsPsych.randomization.sampleWithoutReplacement()`来确保实验条件的随机分配。

**问题5：如何处理跨浏览器的兼容性问题？**

-   解决方案：在多个浏览器上测试实验，确保实验在所有目标浏览器上都能正常运行。使用现代、广泛支持的JavaScript和CSS特性。
